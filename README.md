# Polymarket AI

Інструмент для збору даних про ринки прогнозів з [Polymarket](https://polymarket.com) та пошуку торгових сигналів за допомогою LLM (Groq).

## Що робить

1. **Збирає** ринки/події з публічного Gamma API Polymarket (`https://gamma-api.polymarket.com`), посторінково, до кількох тисяч подій.
2. **Валідує та нормалізує** отримані дані (перевірка обов'язкових полів, цін, обсягу).
3. **Відсіює** ринки без дати завершення, ті що вже закінчились, надто віддалені за дедлайном (після 31.12.2026) та дублікати за заголовком.
4. **Аналізує** ринки, що залишились, через Groq (`llama-3.3-70b-versatile`, безкоштовний тір): модель оцінює ймовірність події та рахує математичне очікування (EV) для ставок YES/NO.
5. **Виводить у лог** знайдені торгові сигнали (`BUY_YES` / `BUY_NO`) з ROI, EV та обґрунтуванням — якщо AI-ключ не задано, крок аналізу просто пропускається.

Це дослідницький/аналітичний скрипт, а не торговий бот — реальних угод він не здійснює.

## Структура проекту

```
polymarket_ai/
├── main.py                     # точка входу CLI: збір → валідація → AI-аналіз
├── config.py                   # налаштування (URL API, таймаути, пороги валідації)
├── collectors/
│   ├── api_client.py           # HTTP-клієнт до Gamma API
│   └── market_fetcher.py       # отримання та посторінковий збір ринків/подій
├── processing/
│   ├── normalizer.py           # приведення сирих даних до єдиного формату
│   ├── validators.py           # перевірка даних ринку
│   └── ai_analyzer.py          # запит до Groq та розбір торгового сигналу
├── models/market.py             # модель даних ринку
├── utils/logger.py              # логування
└── webapp/                      # веб-версія (Flask) — див. розділ нижче
    ├── app.py
    └── static/                  # index.html / style.css / app.js
```

## Встановлення

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt        # для розробки: requirements-dev.txt
```

Альтернативно — як editable-пакет (дає команди `polymarket-ai` / `polymarket-ai-web` замість `python -m ...`):

```bash
pip install -e .
```

## Налаштування

Скопіюй `.env.example` в `.env` і за потреби відредагуй:

```bash
cp .env.example .env
```

| Змінна              | Призначення                              | За замовчуванням                     |
|---------------------|--------------------------------------------|----------------------------------------|
| `POLYMARKET_API_URL`| Базовий URL Gamma API                      | `https://gamma-api.polymarket.com`     |
| `API_TIMEOUT`       | Таймаут запитів, сек                       | `10`                                    |
| `API_RETRIES`       | Кількість повторів при збої мережі         | `3`                                     |
| `DEBUG_MODE`        | Детальні логи                              | `false`                                 |
| `LOG_LEVEL`         | Рівень логування                           | `INFO`                                  |
| `LOG_JSON`          | Один лог-рядок = один JSON-об'єкт (зручно для збору логів у Docker/production) | `false` |
| `GROQ_API_KEY`      | Ключ Groq для AI-аналізу (без нього — аналіз пропускається) | — |

`GROQ_API_KEY` читається з `.env` автоматично (`config.py` викликає `load_dotenv()`) —
достатньо вписати ключ у файл `.env`, окремий `export` не потрібен. Безкоштовний ключ
можна створити на [console.groq.com/keys](https://console.groq.com/keys). Альтернативно
його можна ввести прямо у веб-інтерфейсі (⚙ Налаштування) — тоді він зберігається лише
в браузері й має пріоритет над значенням з `.env`.

## Запуск

```bash
python -m polymarket_ai.main
```

Скрипт збере відкриті події, відфільтрує їх за датою і (якщо задано ключ) прожене через AI, після чого виведе в лог статистику та знайдені сигнали.

## Веб-версія

Той самий збір/валідацію/AI-аналіз можна використовувати через локальний веб-інтерфейс
замість читання логів у терміналі.

```bash
python -m polymarket_ai.webapp.app
```

Відкрий у браузері `http://127.0.0.1:5050` (порт можна змінити змінною `WEB_PORT`).

Юзкейси веб-версії:

- **Огляд ринків (Explorer)** — пошук за назвою, фільтр за категорією/мін. обсягом/терміном
  до закриття, сортування, пагінація. Дані кешуються на 5 хв, кнопка «⟳ Оновити» форсує
  повторний фетч з Gamma API.
- **Скоро закриються** — окремий радар ринків, що резолвляться в найближчі 6 год / 24 год /
  3 дні / 7 днів, відсортований за часом до закриття.
- **AI-аналіз за клацанням** — кнопка «🤖 AI-аналіз» на будь-якій картці ринку рахує сигнал
  (`BUY_YES` / `BUY_NO` / `HOLD`), ROI, EV та обґрунтування без потреби ганяти аналіз по
  всіх тисячах ринків одразу, як у CLI.
- **Сканування топ-10** — кнопка «🔎 Сканувати топ-10 на AI-сигнали» проганяє AI по перших
  10 ринках поточної вибірки (з урахуванням фільтрів) і показує тільки ті, де знайдено
  реальний сигнал (не `HOLD`). Обмежено 15 ринками за раз, щоб не впертись у ліміти
  безкоштовного тіру Groq.
- **Список спостереження (Watchlist)** — зірочка ☆ на картці зберігає ринок у локальний
  файл `polymarket_ai/webapp/watchlist.json` (не входить у git), окрема вкладка показує
  всі збережені ринки.
- **Експорт CSV** — кнопка «⬇ Експорт CSV» вивантажує поточну відфільтровану вибірку
  прямо в браузері, без додаткового запиту до сервера.
- **Налаштування (⚙)** — Groq ключ вводиться прямо в браузері (зберігається лише в
  `localStorage` цього браузера, передається на сервер тільки в заголовку запиту, ніколи
  не записується у файли проєкту) — зручно, якщо не хочеш возитись з `export GROQ_API_KEY`
  у терміналі щоразу.

## Docker

```bash
export GROQ_API_KEY="gsk_..."          # опційно
touch polymarket_ai/webapp/watchlist.json polymarket_ai/webapp/translations.json
docker compose up --build
```

Відкрий `http://127.0.0.1:5050`. `watchlist.json` і `translations.json` монтуються як
bind mount — файли мають існувати на хості *до* першого запуску (Docker інакше створить
на їх місці директорію).
